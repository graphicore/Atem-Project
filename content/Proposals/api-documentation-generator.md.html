API Documentation Generator

This is a stub!

Documentation is a need that we have. This is a navigational dead end to raise awareness for our need of an automated system to generate well structured API documentation from our original source code, where then as well the source codes will have to be changed to feed into that system.

What we want is to have good readable and ideally hyper-linked documentation generated from our original source files. In the best case synced via git hook and travis on update or on new tag releases.

Generating the docs has higher priority than automatic updating, in fact, the latter could be impractical.

Maybe we can do or reuse something in the spirit of literate programming.
